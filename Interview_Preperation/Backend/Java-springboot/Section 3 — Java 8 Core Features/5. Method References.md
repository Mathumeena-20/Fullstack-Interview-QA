# 1. What is a Method Reference?

A **method reference** is a shorthand notation of a lambda expression that calls an existing method.

### Example:

**Lambda:**

```java
list.forEach(s -> System.out.println(s));
```

**Method reference:**

```java
list.forEach(System.out::println);
```

‚úî More concise
‚úî More readable
‚úî No need to rewrite existing methods

---

# 2. Types of Method References

Java provides **three types**:

---

## üîπ 1. Static method reference ‚Äî `Class::staticMethod`

### Example:

```java
class Utils {
    public static void print(String s) {
        System.out.println(s);
    }
}

list.forEach(Utils::print);
```

---

## üîπ 2. Instance method reference ‚Äî `object::instanceMethod`

### Example:

```java
String prefix = "Hello ";

Function<String, String> func = prefix::concat;
System.out.println(func.apply("World")); // Hello World
```

---

## üîπ 3. Constructor reference ‚Äî `Class::new`

### Example:

```java
Supplier<List<String>> listSupplier = ArrayList::new;

List<String> list = listSupplier.get();
```

---

# 3. Difference between Lambda Expression and Method Reference

| Aspect      | Lambda                   | Method Reference                    |
| ----------- | ------------------------ | ----------------------------------- |
| Style       | Custom logic             | Calls an existing method            |
| Readability | Sometimes verbose        | Very clean & short                  |
| Flexibility | Can contain statements   | Only calls method                   |
| Use case    | When custom logic needed | When existing method fits signature |

### Example:

```java
// Lambda
Comparator<String> c1 = (a, b) -> a.compareToIgnoreCase(b);

// Method reference
Comparator<String> c2 = String::compareToIgnoreCase;
```

---

# 4. When Should You Prefer Method References?

‚úî When lambda only calls an existing method
‚úî When readability improves
‚úî When using utility/helper methods
‚úî When working with Streams & functional interfaces

---

# 5. Can Method References Replace All Lambdas?

‚ùå No.

Use lambda when:

* You need multiple statements
* You add additional logic
* You need local variables or conditions

Example (cannot use method reference):

```java
list.forEach(s -> {
    if (s.startsWith("A"))
        System.out.println(s);
});
```

---

# 6. How Does Method Reference Bind Arguments?

Arguments are passed automatically based on the functional interface method.

### Example:

```java
Function<String, Integer> func = Integer::parseInt;
```

* Functional method: `apply(String)`
* Method reference: `parseInt(String)`
* JVM binds that `apply()` ‚Üí calls `parseInt()`

# 7. Real-Time Use Cases of Method References

### 1. Logging

```java
list.forEach(Logger::info);
```

### 2. Stream chaining

```java
list.stream()
    .map(String::toUpperCase)
    .forEach(System.out::println);
```

### 3. Constructor injection

```java
Supplier<Employee> emp = Employee::new;
```

### 4. Sorting

```java
list.sort(String::compareToIgnoreCase);
```

---

# 8. What Happens If Method Signature Doesn‚Äôt Match?

‚ùå Compile-time error.

Example:

```java
Function<String, Integer> f = String::length;  // OK
Function<String, Integer> f2 = String::toUpperCase;  // ‚ùå WRONG
```

Why wrong?
`Integer apply(String)` expects return type Integer, but `toUpperCase()` returns String.

---

# 9. Can Private Methods Be Used in Method References?

Yes, **within the same class**.

Example:

```java
class Test {
    private void show() {
        System.out.println("Private method");
    }

    void call() {
        Runnable r = this::show;
        r.run();
    }
}
```

‚úî Method references follow normal access rules.

---

# 10. Method Reference vs Stream Operations

Method references are often used inside Streams:

### Example:

```java
list.stream()
    .map(String::trim)
    .filter(String::isEmpty)
    .forEach(System.out::println);
```

Streams = processing pipeline
Method references = cleaner syntax for stream functions

They work **together**, not against each other.
