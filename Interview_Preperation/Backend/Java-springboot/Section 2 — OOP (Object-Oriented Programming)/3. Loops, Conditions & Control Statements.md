## 1. Difference between `for`, `while`, and `do-while`

| Feature                 | `for` loop              | `while` loop         | `do-while` loop        |
| ----------------------- | ----------------------- | -------------------- | ---------------------- |
| Condition check         | Before execution        | Before execution     | After execution        |
| Executes at least once? | ❌ No                    | ❌ No                 | ✅ Yes                  |
| Best used when          | No. of iterations known | Condition based loop | Must run at least once |

### Example:

```java
// for loop
for(int i = 1; i <= 3; i++) {
    System.out.println(i);
}

// while loop
int j = 1;
while(j <= 3) {
    System.out.println(j);
    j++;
}

// do-while loop
int k = 5;
do {
    System.out.println(k);
    k++;
} while(k <= 3);   // Prints once
```

## 2. When would you prefer `do-while` over `while`?

When the loop must execute **at least once**, like:

✅ Menu-driven programs
✅ Taking user input until valid
✅ Game loops

### Example:

```java
int choice;
do {
    System.out.println("Enter 1 to exit:");
    choice = scanner.nextInt();
} while(choice != 1);
```

## 3. What is an Enhanced for loop? Limitations?

Also called **for-each loop**.

### Syntax:

```java
for(datatype var : collection) {
   // code
}
```

### Example:

```java
int[] nums = {10, 20, 30};
for(int num : nums) {
    System.out.println(num);
}
```

### ✅ Advantages:

✔ More readable
✔ No index handling

### ❌ Limitations:

❌ No access to index
❌ Cannot traverse backward
❌ Cannot modify collection safely

## 4. Difference between `break` and `continue`

| break              | continue                |
| ------------------ | ----------------------- |
| Terminates loop    | Skips current iteration |
| Jumps outside loop | Jumps to next iteration |

### Example:

```java
for(int i = 1; i <= 5; i++) {
    if(i == 3) break;
    System.out.print(i + " ");
}
// Output: 1 2

for(int i = 1; i <= 5; i++) {
    if(i == 3) continue;
    System.out.print(i + " ");
}
// Output: 1 2 4 5
```

## 5. What is a Labeled Break?

Used to break a specific outer loop.

### Example:

```java
outer:
for(int i = 1; i <= 3; i++) {
    for(int j = 1; j <= 3; j++) {
        if(j == 2) break outer;
        System.out.println(i + "," + j);
    }
}
```

Output:

```
1,1
```

## 6. Can we use conditions inside a `switch`?

❌ Relational conditions are not allowed.

Allowed:

```java
int x = 5;
switch(x){
    case 5:
        System.out.println("Five");
}
```

Not allowed:

```java
case x > 5:   // ❌ Not allowed
```

## 8. How does JVM process String in switch?

Example:

```java
String day = "MON";

switch(day){
    case "MON":
        System.out.println("Monday");
        break;
}
```

### Internals:

1. JVM converts each case String to `hashCode()`
2. If hash matches, then calls `equals()` for confirmation.

## 9. What is Fall-Through in switch?

When `break` is missing → control passes to next case.

### Example:

```java
int x = 1;
switch(x){
    case 1:
        System.out.println("One");
    case 2:
        System.out.println("Two");
}
```

### Output:

```
One
Two
```

## 10. If-else vs Ternary Operator

| if-else             | Ternary operator     |
| ------------------- | -------------------- |
| Used for logic      | Used for expressions |
| Longer code         | Shorter code         |
| No return guarantee | Returns a value      |

### Example:

```java
int a = 10, b = 20;

// if-else
int max;
if(a > b) max = a;
else max = b;

// ternary
int max2 = (a > b) ? a : b;
```
