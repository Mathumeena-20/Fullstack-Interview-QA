## 1. What is a Package in Java?

A **package** is a namespace that groups related classes and interfaces.
It helps in organizing large applications and avoiding class name conflicts.

### Example:

```java
package com.company.project;

public class Employee {
    public void show() {
        System.out.println("Employee class");
    }
}
```

---

## 2. Difference Between Package and Module (Java 9+)

| Package                           | Module                                       |
| --------------------------------- | -------------------------------------------- |
| Groups related classes/interfaces | Groups related packages                      |
| Available since early Java        | Introduced in Java 9                         |
| Organizational purpose            | Provides strong encapsulation & dependencies |
| No dependency control             | Controls what packages are exported          |

### Example:

```java
// module-info.java
module com.company.project {
    exports com.company.project;
}
```

---

## 3. Why are Packages Used?

✅ Avoid name conflicts
✅ Code organization
✅ Access control
✅ Improves maintainability and scalability

---

## 4. Difference Between `public`, `default` and `protected`

| Modifier             | Same Class | Same Package | Subclass (different package) | Outside Package          |
| -------------------- | ---------- | ------------ | ---------------------------- | ------------------------ |
| public               | ✅          | ✅            | ✅                            | ✅                        |
| default (no keyword) | ✅          | ✅            | ❌                            | ❌                        |
| protected            | ✅          | ✅            | ✅                            | ❌ (only via inheritance) |

## 5. What is Classpath?

The **classpath** tells JVM where to find:

* Class files
* JAR files
* Libraries

### Example:

```bash
java -cp /myclasses:/libs/app.jar Main
```

---

## 6. Can a Package Have Sub-Packages?

✅ Yes.

Example hierarchy:

```
com.company
com.company.project
com.company.project.service
```

---

## 7. Difference Between `import` and `static import`

### Normal `import`

Imports classes or interfaces.

```java
import java.util.List;
```

### `static import`

Imports static members like methods/variables.

```java
import static java.lang.Math.PI;

System.out.println(PI);
```

---

## 8. How Does JVM Locate a Class?

Given:

```java
package com.example.service;
```

Directory must be:

```
com/example/service/ClassName.class
```

JVM checks:

1. Classpath
2. Folder structure
3. Package name

---

## 9. Can a Class Belong to Multiple Packages?

❌ No.
A class can belong to only one package.

---

## 10. How to Create a Custom Package?

### Step 1 – Write code:

```java
package mypackage;

public class Demo {
    public void display() {
        System.out.println("Hello from custom package");
    }
}
```

### Step 2 – Compile:

```bash
javac -d . Demo.java
```

This creates:

```
mypackage/Demo.class
```

---

### Step 3 – Use the package:

```java
import mypackage.Demo;

public class Main {
    public static void main(String[] args) {
        Demo d = new Demo();
        d.display();
    }
}
```
