## 1. What does the `static` keyword mean?

`static` means the member (variable/method/block) belongs to the **class**, not the object.

✅ Only **one copy** exists in memory
✅ Shared by all objects of the class
✅ Loaded when the class is loaded

### Example:

```java
class Student {
    static String college = "MIT";  // shared
    String name;                    // unique per object
}
```

---

## 2. When is a static block executed?

A static block is executed:
✔ Once
✔ When the class is loaded into JVM
✔ Before `main()` executes

### Example:

```java
class Test {
    static {
        System.out.println("Static block executed");
    }

    public static void main(String[] args) {
        System.out.println("Main executed");
    }
}
```

### Output:

```
Static block executed
Main executed
```

## 3. Static Variable vs Instance Variable

| Feature    | Static Variable         | Instance Variable         |
| ---------- | ----------------------- | ------------------------- |
| Belongs to | Class                   | Object                    |
| Memory     | Single copy             | Separate per object       |
| Access     | ClassName.variable      | object.variable           |
| Lifetime   | Until class is unloaded | Until object is destroyed |

### Example:

```java
class Test {
    static int count = 0;  // static
    int id;                 // instance variable
}
```

---

## 4. Can we access a non-static variable in a static method?

❌ Direct access is not allowed
✅ Indirect with object reference

### Example:

```java
class Demo {
    int num = 10;

    static void show() {
        // System.out.println(num); ❌ Error
        Demo obj = new Demo();
        System.out.println(obj.num);  // ✅
    }
}
```

---

## 5. Can a class be static?

❌ A top-level class CANNOT be static
✅ A nested class CAN be static

### Example:

```java
class Outer {
    static class Inner {
        void display() {
            System.out.println("Static inner class");
        }
    }
}
```

---

## 6. Why is the `main()` method static?

Because JVM runs `main()` without creating an object.

JVM does not know how to create an object before executing `main()`.

---

## 7. Execution order: static & instance blocks

When both are present:

1️⃣ Static block executes first (once)
2️⃣ Instance block executes next (every object)
3️⃣ Constructor executes last

### Example:

```java
class Test {
    static {
        System.out.println("Static block");
    }

    {
        System.out.println("Instance block");
    }

    Test() {
        System.out.println("Constructor");
    }

    public static void main(String[] args) {
        Test t1 = new Test();
    }
}
```

### Output:

```
Static block
Instance block
Constructor
```

## 8. Static vs Instance Method

| Feature           | Static Method | Instance Method |
| ----------------- | ------------- | --------------- |
| Call using        | Class name    | Object          |
| Access non-static | ❌ No          | ✅ Yes           |
| Belongs to        | Class         | Object          |
| Overriding        | ❌ No          | ✅ Yes           |

---

## 9. Can static methods be overridden?

❌ Static methods cannot be overridden
They are **method hidden**, not overridden.

### Example:

```java
class Parent {
    static void show() {
        System.out.println("Parent");
    }
}

class Child extends Parent {
    static void show() {
        System.out.println("Child");
    }
}

Parent obj = new Child();
obj.show();   // Output: Parent
```

---

## 10. Where are static variables stored in JVM?

Static variables are stored in:

✅ **Method Area** (Java 7 and below)
✅ **Metaspace** (Java 8 and above)

They exist as long as the class is loaded.
