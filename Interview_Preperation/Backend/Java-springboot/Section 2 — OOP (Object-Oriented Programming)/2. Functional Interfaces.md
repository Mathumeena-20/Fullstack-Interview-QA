## 1. Primitive vs Non-Primitive Data Types

### üîπ Primitive Data Types

Store **actual values directly in memory**.

| Type    | Size          | Example                |
| ------- | ------------- | ---------------------- |
| byte    | 1 byte        | `byte b = 10;`         |
| short   | 2 bytes       | `short s = 100;`       |
| int     | 4 bytes       | `int n = 500;`         |
| long    | 8 bytes       | `long l = 9999L;`      |
| float   | 4 bytes       | `float f = 10.5f;`     |
| double  | 8 bytes       | `double d = 100.99;`   |
| char    | 2 bytes       | `char c = 'A';`        |
| boolean | JVM dependent | `boolean flag = true;` |

---

### üîπ Non-Primitive Data Types

Store **references (address)** to objects.

Examples:
`String`, `Arrays`, `Classes`, `Interfaces`, `Objects`

Example:

```java
String name = "Java";
int[] arr = {1, 2, 3};
```

## 2. Difference Between `int` and `Integer`

| Feature              | int       | Integer       |
| -------------------- | --------- | ------------- |
| Type                 | Primitive | Wrapper class |
| Can be null?         | ‚ùå No      | ‚úÖ Yes         |
| Stored in            | Stack     | Heap          |
| Used in Collections? | ‚ùå No      | ‚úÖ Yes         |
| Default value        | 0         | null          |

### Example:

```java
int a = 5;
Integer b = 5;

List<Integer> list = new ArrayList<>();
list.add(a);  // Autoboxing
```

## 3. Why is `char` 2 bytes in Java?

Java uses **Unicode** (UTF-16), not ASCII.

Unicode characters need **2 bytes (16 bits)**.

Example:

```java
char c = '‡§Æ';  // Works because Java supports Unicode
System.out.println(c);
```

## 4. Type Casting: Implicit vs Explicit

### ‚úÖ Implicit (Widening)

Smaller type ‚Üí Bigger type.

```java
int a = 10;
double b = a;  // Automatically converted
```

---

### ‚úÖ Explicit (Narrowing)

Bigger type ‚Üí Smaller type.

```java
double x = 10.5;
int y = (int) x;   // Output: 10
```

---

## 5. Autoboxing and Unboxing

### Autoboxing

Primitive ‚Üí Wrapper.

```java
int a = 5;
Integer obj = a;
```

### Unboxing

Wrapper ‚Üí Primitive.

```java
Integer obj = 10;
int b = obj;
```

## 6. Default Values of Primitive Data Types

| Data Type | Default  |
| --------- | -------- |
| byte      | 0        |
| short     | 0        |
| int       | 0        |
| long      | 0L       |
| float     | 0.0f     |
| double    | 0.0      |
| char      | '\u0000' |
| boolean   | false    |

Example:

```java
class Test {
    int num;
    boolean flag;

    public static void main(String[] args) {
        Test t = new Test();
        System.out.println(t.num);    // 0
        System.out.println(t.flag);   // false
    }
}
```

## 7. How is `boolean` Stored Internally?

JVM does not define exact size.
Mostly it's stored as **1 byte or 1 bit** internally.

true ‚Üí 1
false ‚Üí 0

---

## 8. Can We Store Very Large Integers?

No, `long` has a limit.

Max value:

```
9,223,372,036,854,775,807
```

For bigger numbers ‚Üí Use `BigInteger`.

Example:

```java
import java.math.BigInteger;

BigInteger num = new BigInteger("9999999999999999999999");
System.out.println(num.add(BigInteger.TWO));
```

## 9. Why Java is Not Fully Object-Oriented?

Because Java has **primitive data types** like:
`int`, `char`, `boolean` etc.

In fully OOP languages, everything is an object.

---

## 10. What is a Wrapper Class & Its Importance?

Wrapper classes convert primitives ‚Üí objects.

| Primitive | Wrapper   |
| --------- | --------- |
| int       | Integer   |
| char      | Character |
| double    | Double    |
| boolean   | Boolean   |

### Why important?

‚úÖ Needed for Collections
‚úÖ Supports object features
‚úÖ Supports generics
‚úÖ Allows null values

### Example:

```java
List<Integer> list = new ArrayList<>();
list.add(10);   // Autoboxing happens
```
