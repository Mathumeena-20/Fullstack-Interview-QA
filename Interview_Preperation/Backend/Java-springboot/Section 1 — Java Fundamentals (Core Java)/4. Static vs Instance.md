## 1. What does the `static` keyword mean?

`static` belongs to the **class**, not to the object.

* Only **one copy** is shared among all objects.
* Memory is allocated **once** when class is loaded.

### Example:

```java
class Student {
    static String college = "ABC";
    String name;
}
```

All students share the same `college`.

---

## 2. When is a static block executed?

✅ Static blocks run **once**, when the class is loaded by JVM.

### Example:

```java
class Test {
    static {
        System.out.println("Static block executed");
    }

    public static void main(String[] args) {
        System.out.println("Main method executed");
    }
}
```

### Output:

```
Static block executed
Main method executed
```

## 3. Difference between static and instance variables

| Feature        | Static Variable    | Instance Variable        |
| -------------- | ------------------ | ------------------------ |
| Belongs to     | Class              | Object                   |
| Memory         | One copy           | Separate for each object |
| Accessed using | Classname.variable | Object reference         |
| Lifetime       | Till program ends  | Till object exists       |

### Example:

```java
class Example {
    static int a = 10;
    int b = 20;
}
```

---

## 4. Can we access a non-static variable inside a static method?

❌ No, because static methods don’t belong to an object.

But we can access using object reference:

### Example:

```java
class Test {
    int x = 10;

    static void show() {
        // System.out.println(x); ❌ Error
        Test obj = new Test();
        System.out.println(obj.x); ✅
    }
}
```

---

## 5. Can a class be static?

❌ A top-level class cannot be static.
✅ But nested classes can be static.

### Example:

```java
class Outer {
    static class Inner {
        void display() {
            System.out.println("Static nested class");
        }
    }
}
```

---

## 6. Why is `main()` method static?

Because JVM calls `main()` **without creating an object**.

If it wasn’t static, JVM would need object first which is not possible.

---

## 7. What happens when both static and instance blocks are present?

Execution order:

1. Static block
2. Instance block
3. Constructor

### Example:

```java
class Demo {
    static {
        System.out.println("Static block");
    }

    {
        System.out.println("Instance block");
    }

    Demo() {
        System.out.println("Constructor");
    }

    public static void main(String[] args) {
        Demo d = new Demo();
    }
}
```

### Output:

```
Static block
Instance block
Constructor
```

## 8. Static method vs Instance method

| Feature                      | Static method | Instance method |
| ---------------------------- | ------------- | --------------- |
| Called using                 | Class name    | Object          |
| Access non-static variables? | ❌ No          | ✅ Yes           |
| Belongs to                   | Class         | Object          |

### Example:

```java
class Test {
    static void display() {
        System.out.println("Static method");
    }

    void show() {
        System.out.println("Instance method");
    }
}
```

---

## 9. Can static methods be overridden?

❌ Static methods **cannot be overridden**.
They are **method hidden**, not overridden.

### Example:

```java
class Parent {
    static void show() {
        System.out.println("Parent show");
    }
}

class Child extends Parent {
    static void show() {
        System.out.println("Child show");
    }
}

Parent obj = new Child();
obj.show();  // Output: Parent show
```

---

## 10. Where are static variables stored in JVM?

Static variables are stored in:

* **Method Area** (older JVM)
* In modern JVMs: **Metaspace**

They exist till the class is unloaded.
