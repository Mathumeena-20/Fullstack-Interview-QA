## 1. What is a Constructor?

A **constructor** is a special method used to **initialize objects**.

### Characteristics:

* Same name as the class
* No return type (not even `void`)
* Called automatically when object is created

### Example:

```java
class Student {
    String name;

    Student(String name) {
        this.name = name;
    }
}
```

---

## 2. Difference between Constructor and Method

| Constructor          | Method                 |
| -------------------- | ---------------------- |
| Initializes object   | Performs operations    |
| Same name as class   | Any valid method name  |
| No return type       | Has return type / void |
| Called automatically | Called manually        |
| Runs once per object | Can run multiple times |

---

## 3. Types of Constructors

### 1. Default Constructor

Created by compiler if no constructor is defined.

```java
class Car {
    int speed;
}
Car c = new Car();  // Default constructor
```

---

### 2. Parameterized Constructor

Accepts parameters.

```java
class Car {
    int speed;

    Car(int speed) {
        this.speed = speed;
    }
}
```

---

### 3. No-Arg Constructor

Created by programmer explicitly.

```java
class Bike {
    Bike() {
        System.out.println("No-arg constructor");
    }
}
```

---

## 4. What happens if you don’t define a constructor?

Java compiler automatically provides a **default constructor**:

```java
Bike bike = new Bike();  
```

⚠ But if you define any constructor, compiler will NOT create a default one.

---

## 5. Can constructors be overloaded?

✅ Yes.

Constructor overloading means multiple constructors with different parameters.

### Example:

```java
class Student {
    String name;
    int age;

    Student() {
        name = "Unknown";
    }

    Student(String name) {
        this.name = name;
    }

    Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

---

## 6. Can constructors be overridden?

❌ NO. Constructors cannot be overridden because:

* They are not inherited
* Overriding happens only with methods

---

## 7. What is Constructor Chaining?

Calling one constructor from another constructor using:

* `this()`
* `super()`

### Example:

```java
class Student {
    Student() {
        this("John");  // Call another constructor
    }

    Student(String name) {
        System.out.println("Name: " + name);
    }
}
```

---

## 8. What is `this()` and `super()`?

### this()

* Refers to current object
* Used to call another constructor in same class

### super()

* Refers to parent class object
* Used to call parent constructor

### Example:

```java
class Parent {
    Parent() {
        System.out.println("Parent constructor");
    }
}

class Child extends Parent {
    Child() {
        super();  // Calls parent constructor
        System.out.println("Child constructor");
    }
}
```

### Output:

```
Parent constructor
Child constructor
```

---

## 9. Rules of Method Overloading

1. Must have same method name
2. Different parameter list:

   * Number of parameters
   * Type of parameters
   * Order of parameters
3. Return type alone is **not enough**

---

### ✅ Valid overloading

```java
class Calc {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
}
```

### ❌ Invalid

```java
int add(int a, int b);
double add(int a, int b);  // Error
```

## 10. Can we overload the `main()` method?

✅ Yes, but JVM always calls this one:

```java
public static void main(String[] args)
```

### Example:

```java
class Test {

    public static void main(String[] args) {
        System.out.println("Main with String[]");
        main(10);
    }

    public static void main(int a) {
        System.out.println("Overloaded main: " + a);
    }
}
```

### Output:

```
Main with String[]
Overloaded main: 10
```
