## 1. Difference between `for`, `while`, and `do-while` loops

| Feature                 | for                 | while                         | do-while               |
| ----------------------- | ------------------- | ----------------------------- | ---------------------- |
| Entry control?          | ‚úÖ Yes               | ‚úÖ Yes                         | ‚ùå No (Exit control)    |
| Condition checked       | Before loop starts  | Before loop starts            | After loop body runs   |
| Executes at least once? | ‚ùå                   | ‚ùå                             | ‚úÖ Yes                  |
| Best used when          | You know iterations | Repetition based on condition | Must run at least once |

### Example:

```java
// for loop
for (int i = 1; i <= 3; i++) {
    System.out.println(i);
}

// while loop
int i = 1;
while (i <= 3) {
    System.out.println(i);
    i++;
}

// do-while loop
int j = 5;
do {
    System.out.println(j);
    j++;
} while (j <= 3);
```

üëâ `do-while` prints `5` once even though the condition is false.

## 2. When would you prefer `do-while` over `while`?

When the loop body **must execute at least once**, such as:

* Menu driven programs
* Taking user input until valid
* Retry scenarios

### Example:

```java
int choice;
do {
    System.out.println("Enter 1 to continue:");
    choice = scanner.nextInt();
} while (choice != 1);
```

## 3. What is an Enhanced for loop? What are its limitations?

Also called **for-each loop**.

### Syntax:

```java
for (dataType var : collection) {
    // code
}
```

### Example:

```java
int[] numbers = {10, 20, 30};

for (int num : numbers) {
    System.out.println(num);
}
```

### ‚úÖ Advantages:

‚úî Clean and simple
‚úî No index handling

### ‚ùå Limitations:

‚ùå No access to index
‚ùå Cannot iterate in reverse
‚ùå Cannot modify the collection (like removing elements)

## 4. Difference between `break` and `continue`

| break                     | continue                       |
| ------------------------- | ------------------------------ |
| Stops loop entirely       | Skips current iteration        |
| Control goes outside loop | Control goes to next iteration |

### Example:

```java
for (int i = 1; i <= 5; i++) {
    if (i == 3) break;
    System.out.print(i);  
}
// Output: 1 2


for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;
    System.out.print(i);
}
// Output: 1 2 4 5
```

## 5. What is a Labeled Break?

Allows breaking an **outer loop** instead of just the inner loop.

### Example:

```java
outer:
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (j == 2) break outer;
        System.out.println(i + " " + j);
    }
}
```

üëâ Breaks the outer loop when `j == 2`.

## 6. Can we use conditions inside a `switch`?

Direct relational conditions like `>`, `<` are ‚ùå NOT allowed.

But you can simulate using:

* range checks before switch
* enum or pre-calculated values

### Valid:

```java
int x = 5;

switch (x) {
    case 5:
        System.out.println("Five");
        break;
}
```

### ‚ùå Invalid:

```java
switch(x) {
    case x > 5:  // Not allowed
}
```

## 7. From Java 7+, what changes happened in `switch`?

‚úÖ Support for `String`
‚úÖ Support for `enum`
‚úÖ Improved bytecode handling

Before Java 7: only int, char, byte, short

## 8. Can we use `String` in `switch`? How does JVM process it?

Yes, from Java 7 onwards.

### Example:

```java
String day = "MON";

switch (day) {
    case "MON":
        System.out.println("Monday");
        break;
    case "TUE":
        System.out.println("Tuesday");
        break;
}
```

### Internally JVM:

* Converts each string to `hashCode()`
* Compares using `equals()` if hash matches

## 9. What is Fall-through in switch?

When `break` is missing, the control flows into next case.

### Example:

```java
int x = 1;
switch (x) {
    case 1:
        System.out.println("One");
    case 2:
        System.out.println("Two");
    case 3:
        System.out.println("Three");
}
```

### Output:

```
One
Two
Three
```

## 10. Difference between if-else and Ternary Operator

| Feature        | if-else               | Ternary           |
| -------------- | --------------------- | ----------------- |
| Used for       | Conditions with logic | Simple conditions |
| Syntax         | Long                  | Short             |
| Returns value? | No                    | Yes               |

### Ternary Syntax:

```java
condition ? valueIfTrue : valueIfFalse;
```

### Example:

```java
int a = 10, b = 20;

int max = (a > b) ? a : b;

System.out.println(max);  // 20
```
