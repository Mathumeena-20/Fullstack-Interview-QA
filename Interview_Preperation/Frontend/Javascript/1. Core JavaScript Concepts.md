# ✅ **1. What is the difference between `==` and `===`?**

### **Answer:**

* `==` → compares **values only**, performs **type coercion**.
* `===` → compares **values + types**, **no type coercion**.

### **Example**

```js
1 == "1"    // true  → converts "1" to number
1 === "1"   // false → types are different
true == 1   // true  → true converted to 1
true === 1  // false
```

### **When to use?**

Always prefer **===** to avoid unexpected type conversions.

# ✅ **2. Difference between `var`, `let`, and `const`**

| Feature   | `var`                              | `let`            | `const`          |
| --------- | ---------------------------------- | ---------------- | ---------------- |
| Scope     | Function-scoped                    | Block-scoped     | Block-scoped     |
| Hoisted   | Yes (initialized with `undefined`) | Yes (but in TDZ) | Yes (but in TDZ) |
| Reassign  | Yes                                | Yes              | No               |
| Redeclare | Yes                                | No               | No               |

### **Example**

```js
if (true) {
  var x = 10;
  let y = 20;
}
console.log(x); // 10
console.log(y); // ReferenceError (block scoped)
```

# ✅ **3. Explain hoisting in JavaScript**

**Hoisting** = JavaScript moves **variable and function declarations** to the top of their scope **during the memory creation phase**.

### What gets hoisted?

* `var` → hoisted with **default value = undefined**
* `let` & `const` → hoisted but kept in **Temporal Dead Zone (TDZ)**
* Function declarations → hoisted **completely**

### Example

```js
console.log(a);  // undefined (var hoisted)
var a = 10;

console.log(b);  // ReferenceError (TDZ)
let b = 20;
```

# ✅ **4. What is closure? Give a real example.**

A **closure** occurs when an inner function **remembers variables of its outer function**, even after the outer function is finished.

### Example: Counter function (real use-case)

```js
function createCounter() {
    let count = 0;

    return function () {
        count++;
        console.log(count);
    };
}

const counter = createCounter();
counter(); // 1
counter(); // 2
counter(); // 3
```

`count` lives **inside the closure**, not the global scope.

### Real uses:

* Private variables
* Memoization
* Event handlers
* Module pattern

# ✅ **5. What is the Temporal Dead Zone (TDZ)?**

TDZ is the period between:

1. **Hoisting** of `let`/`const`, and
2. **Their initialization**

You **cannot access them** in this window.

### Example

```js
console.log(x); // ReferenceError → TDZ
let x = 5;
```

# ✅ **6. What is event bubbling and capturing?**

Events travel through the DOM in **three phases**:

### **1) Capturing phase (top → target)**

Browser checks handlers from **document → parent → target**.

### **2) Target phase**

Event reaches the actual element.

### **3) Bubbling phase (target → top)**

Event propagates **back upward**.

### Example

```html
<div id="parent">
  <button id="child">Click Me</button>
</div>
<script>
parent.addEventListener('click', () => console.log("Parent"), true);  // capture
child.addEventListener('click', () => console.log("Child"));          // bubble
</script>
```

Output when clicking button:

```
Parent
Child
```

# ✅ **7. Difference between `for…in` and `for…of`**

| Feature  | `for…in`              | `for…of`                    |
| -------- | --------------------- | --------------------------- |
| Iterates | keys (property names) | values in iterable          |
| Works on | Objects, arrays       | Arrays, strings, maps, sets |
| Use-case | Object traversal      | Looping values              |

### Example

```js
const arr = ["a", "b", "c"];

for (let i in arr) {
  console.log(i);       // 0,1,2 (indexes)
}

for (let val of arr) {
  console.log(val);     // a,b,c (values)
}
```

# ✅ **8. What are pure functions? Why important?**

A **pure function**:

1. Always returns the **same output** for the same input
2. Has **no side effects** (doesn’t modify external data)

### Example

```js
function add(a, b) {
  return a + b; // pure
}
```

### Impure example (side effect):

```js
let x = 10;

function addToX(a) {
  x = x + a; // modifies external variable → bad
}
```

### Why important?

* Predictable
* Testable
* No hidden bugs
* Improves functional programming

# ✅ **9. What is immutability? How do you achieve it?**

**Immutability** = data cannot be modified directly.
Instead, you create a **new copy** with changes.

### Achieve using:

* Spread operator `...`
* `Object.assign`
* `Object.freeze`

### Example: Immutable update

```js
const user = { name: "Mathu", age: 24 };

const updatedUser = { ...user, age: 25 };

console.log(user.age);        // 24
console.log(updatedUser.age); // 25
```

### Using Object.freeze

```js
const obj = Object.freeze({ a: 10 });
obj.a = 20;   // ❌ ignored in strict mode
```

# ✅ **10. What are higher-order functions?**

A **Higher-Order Function (HOF)** is a function that:

✔ Accepts another function as argument
✔ OR returns a function
✔ OR both

### Example: Passing a function

```js
function greet() {
  return "Hello";
}

function say(fn) {
  console.log(fn());
}

say(greet);  // Hello
```

### Example: Returning a function (closure)

```js
function multiplier(x) {
  return function(y) {
    return x * y;
  }
}

const double = multiplier(2);
console.log(double(5)); // 10
```
