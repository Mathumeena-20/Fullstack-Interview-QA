# ‚úÖ **1. How does Flexbox work?**

Flexbox is a one-dimensional layout system used to align items **in a row OR column**.

You define a flex container:

```css
.container {
  display: flex;
}
```

Then Flexbox arranges children based on:

* **Main axis** ‚Üí direction (`flex-direction`)
* **Cross axis** ‚Üí perpendicular to main axis

### Example

```css
.container {
  display: flex;
  flex-direction: row; /* default */
}
```

Flexbox is mainly for **horizontal or vertical alignment‚Äînot both at same time**.

# ‚úÖ **2. Difference between `justify-content` and `align-items`**

| Property          | Aligns items along               | Works on              |
| ----------------- | -------------------------------- | --------------------- |
| `justify-content` | **Main axis** (row ‚Üí horizontal) | Space between items   |
| `align-items`     | **Cross axis** (row ‚Üí vertical)  | Single line alignment |

### Example

```css
.container {
  display: flex;
  justify-content: center; /* horizontal center */
  align-items: center;      /* vertical center */
}
```

# ‚úÖ **3. What does `flex: 1` mean?**

`flex: 1` is shorthand for:

```
flex-grow: 1;
flex-shrink: 1;
flex-basis: 0;
```

Meaning:

* Grow to fill available space
* Shrink if needed
* Start with no initial width

### Example

```css
.item {
  flex: 1; /* all items same flexible width */
}
```

---

# ‚úÖ **4. Explain flex-grow, flex-shrink, and flex-basis**

### **flex-grow**

How much an item expands when there is **extra space**.

Example:

```css
.item1 { flex-grow: 1; }
.item2 { flex-grow: 2; }  /* grows twice as fast */
```

### **flex-shrink**

How much an item shrinks when space is **too small**.

```css
.item1 { flex-shrink: 1; }
.item2 { flex-shrink: 0; } /* never shrink */
```

### **flex-basis**

Initial size **before** growing/shrinking.

```css
.item { flex-basis: 200px; }
```

# ‚úÖ **5. When should you use CSS Grid over Flexbox?**

### Use **Flexbox** when:

* One dimension (row OR column)
* Aligning items
* Navigation bars
* Button groups

### Use **Grid** when:

* Two-dimensional layout (rows **and** columns)
* You need full control of layout structure
* Complex dashboards, page layouts
* Overlapping items

### Simple rule:

‚ùó **Flexbox = 1D**
‚ùó **Grid = 2D**

# ‚úÖ **6. What do grid-template-columns and grid-template-rows do?**

These define the number and size of columns/rows in the grid.

### Example

```css
.container {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: 100px auto;
}
```

Meaning:

* 3 columns ‚Üí 200px, flexible, flexible
* 2 rows ‚Üí 100px and auto height

### fr unit

`fr` = fraction of available space.

# ‚úÖ **7. How to create a responsive grid without media queries?**

Use **auto-fit**, **auto-fill**, and **minmax()**.

### Example (most common pattern)

```css
.container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 20px;
}
```

### How it works:

* `minmax(200px, 1fr)` ‚Üí each item minimum 200px
* `auto-fit` ‚Üí fits as many boxes per row as possible
* Resize browser ‚Üí layout automatically adjusts
  (1, 2, 3, 4 items per row depending on width)

No media queries needed üöÄ
