# âœ… **1. How do you declare CSS variables?**

CSS variables are declared using **custom properties**:

```
--variable-name: value;
```

They are normally defined inside `:root` to make them global.

### **Example**

```css
:root {
  --primary-color: #4CAF50;
  --font-size-large: 20px;
}
```

### **Using a CSS variable**

```css
.button {
  color: var(--primary-color);
  font-size: var(--font-size-large);
}
```

# âœ… **3. How to apply themes using CSS variables?**

You create **different sets of variables** and apply them using classes, such as `.light` and `.dark`.

### Example: Dark / Light theme

```css
:root {
  --bg-color: white;
  --text-color: black;
}

.dark-theme {
  --bg-color: #111;
  --text-color: #eee;
}
```

### Apply to UI

```css
body {
  background: var(--bg-color);
  color: var(--text-color);
}
```

### Toggling theme with JS

```js
document.body.classList.toggle("dark-theme");
```

âœ” Background & text color update instantly (runtime).

# âœ… **4. How to override CSS variables inside components?**

CSS variables **cascade**, so they can be overridden inside *any child selector or component*.

### Example

```css
:root {
  --card-bg: #f5f5f5;
  --card-padding: 20px;
}

.card {
  background: var(--card-bg);
  padding: var(--card-padding);
}
```

### Override the variable for one component

```css
.card.special {
  --card-bg: #ffe8a1;
  --card-padding: 30px;
}
```

Now only `.card.special` uses the new values.

---

### Another Example: Component-level variable overrides

```css
.button {
  --button-color: blue;
  background: var(--button-color);
}

.button.danger {
  --button-color: red;
}
```
