# ✅ **1. What is the CSS Box Model? Explain margin, border, padding, content**

Every HTML element is a **box** consisting of:

1. **Content** → Text / image inside the element
2. **Padding** → Space between content & border
3. **Border** → Line surrounding padding
4. **Margin** → Space outside the border (distance from other elements)

### **Example**

```css
.box {
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  margin: 30px;
}
```

### **How width is calculated**

```
Total Width = content + padding (L+R) + border (L+R) + margin (L+R)
            = 200 + 40 + 10 + 60 = 310px
```

# ✅ **2. Difference between class selectors and id selectors**

| Feature     | Class (`.class`)        | ID (`#id`)                  |
| ----------- | ----------------------- | --------------------------- |
| Uniqueness  | Can be reused           | Must be unique              |
| Specificity | Low                     | High                        |
| Use case    | Style multiple elements | Identify one unique element |
| Syntax      | `.btn`                  | `#header`                   |

### Example

```css
.button { color: blue; }     /* class */
#login-btn { color: red; }   /* id */
```

# ✅ **3. What is specificity? How is it calculated?**

Specificity decides **which CSS rule wins** when multiple rules target the same element.

Specificity is calculated as:

```
Inline styles       → 1000
ID selectors        → 100
Class/attribute/pseudo-class → 10
Element/pseudo-element → 1
```

### Example

```css
#title {}        /* 100 */
.title {}        /* 10 */
h1 {}            /* 1 */
```

### Which wins?

```css
h1.title#main { }   /* 1 + 10 + 100 = 111 → highest */
```

# ✅ **4. Difference between `display: none` and `visibility: hidden`**

| Property             | What it does                           | Layout space      |
| -------------------- | -------------------------------------- | ----------------- |
| `display: none`      | Removes element from layout completely | No space taken    |
| `visibility: hidden` | Element is invisible but still exists  | Space is reserved |

### Example

```css
.box1 { display: none; }
.box2 { visibility: hidden; }
```

# ✅ **5. Difference between block, inline, and inline-block**

### **Block**

* Starts on new line
* Takes full width
* Height/width can be set

Examples: `div`, `p`, `h1`

### **Inline**

* No new line
* Only takes content width
* Width/height cannot be set

Examples: `span`, `a`

### **Inline-block**

* Behaves inline (same line)
* **Width/height can be set**

### Example

```css
.block { display: block; }
.inline { display: inline; }
.inlineBlock { display: inline-block; }
```

# ✅ **6. Difference between position: relative, absolute, fixed, sticky**

### **relative**

* Positioned relative to its **original position**
* Doesn’t remove from layout

### **absolute**

* Removed from layout
* Positioned relative to **nearest positioned ancestor**
* If none → relative to browser window

### **fixed**

* Stays fixed even on scroll
* Relative to viewport

### **sticky**

* Acts like relative
* When scrolled past threshold, it sticks

### Example

```css
.relative { position: relative; top: 10px; }
.absolute { position: absolute; top: 10px; left: 20px; }
.fixed { position: fixed; bottom: 0; }
.sticky { position: sticky; top: 0; }
```

# ✅ **7. What is z-index and how is stacking context created?**

### **z-index**

Controls the **stacking order** (which element appears on top).

Higher `z-index` → element appears above lower values.

### **Stacking context is created when:**

* `position` is `relative | absolute | fixed` **and** `z-index` is set
* `opacity < 1`
* `transform` is applied
* `filter` is applied
* `isolation: isolate`
* `position: sticky` in some scenarios

### Example

```css
.box1 { position: relative; z-index: 10; }
.box2 { position: relative; z-index: 5; }
```

# ✅ **8. Difference between em, rem, px, %**

### **px**

* Absolute unit
* Doesn’t scale with parent or root

### **em**

* Relative to **parent’s font-size**

Example:

```css
.parent { font-size: 20px; }
.child { font-size: 2em; }   /* 40px */
```

### **rem**

* Relative to **root (`html`) font-size**

```css
html { font-size: 16px; }
.box { font-size: 2rem; }  /* 32px */
```

### **%**

* Relative to **parent element** (depends on property)

Example:

```css
.parent { width: 400px; }
.child { width: 50%; }   /* 200px */
```

### Quick Comparison

| Unit | Depends on  | Use-case                     |
| ---- | ----------- | ---------------------------- |
| px   | Fixed pixel | Precise UI                   |
| em   | Parent      | Buttons, nested scaling      |
| rem  | Root        | Global sizing, accessibility |
| %    | Parent      | Layout widths, responsive    |
