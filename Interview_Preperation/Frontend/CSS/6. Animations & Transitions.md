# ‚úÖ **1. How do CSS transitions work?**

CSS transitions animate a property **from its old value ‚Üí new value** when a state change occurs (hover, focus, class change, JS change).

### You need 2 things:

1. **A property that changes**
2. **A transition rule**

### Example

```css
.box {
  width: 100px;
  transition: width 0.4s ease;
}

.box:hover {
  width: 200px;
}
```

‚úî When hovering, width smoothly changes from 100px ‚Üí 200px.

### Transition shorthand

```css
transition: property duration timing-function delay;

transition: all 0.3s ease-in-out;
```

# ‚úÖ **2. Difference between *transition* and *animation***

| Feature           | Transition                  | Animation                                 |
| ----------------- | --------------------------- | ----------------------------------------- |
| Requires trigger? | ‚úî Yes (hover, click, class) | ‚úò No (can run automatically)              |
| Keyframes?        | ‚úò Not used                  | ‚úî Uses @keyframes                         |
| Repeating?        | Only when triggered         | Can loop with `animation-iteration-count` |
| Complexity        | Simple state-to-state       | Multi-step animations                     |

### Example of transition

```css
.box {
  transition: opacity 0.5s;
}
.box:hover {
  opacity: 0.2;
}
```

### Example of animation

```css
.box {
  animation: fade 2s infinite;
}

@keyframes fade {
  0% { opacity: 1; }
  100% { opacity: 0; }
}
```

# ‚úÖ **3. What are keyframes?**

A **keyframe** defines the intermediate states of an animation.

### Example

```css
@keyframes moveRight {
  0%   { transform: translateX(0); }
  50%  { transform: translateX(100px); }
  100% { transform: translateX(0); }
}
```

Usage:

```css
.box {
  animation: moveRight 2s ease infinite;
}
```

‚úî Allows multi-step control
‚úî Used only with CSS animations (not transitions)

# ‚úÖ **4. How does `transform: translate()` differ from `top/left`?**

## **transform: translate()**

* Moves the element **visually** without affecting layout
* Does **not** cause reflow
* Uses GPU acceleration
* Very smooth

```css
.box {
  transform: translateX(50px);
}
```

---

## **top/left**

* Works only when `position: relative / absolute / fixed / sticky`
* Moves the element by changing its **layout position**
* Causes **layout reflow** ‚Üí expensive
* Slower for animations

```css
.box {
  position: absolute;
  left: 50px;
}
```

---

### Visual difference

| Property    | Affects Layout? | Good for Animation? |
| ----------- | --------------- | ------------------- |
| translate() | ‚ùå No            | ‚úî Yes               |
| top/left    | ‚úî Yes (reflow)  | ‚ùå No                |

# ‚úÖ **5. Why are transforms more performant than changing layout properties?**

### ‚úî Reason 1 ‚Äî No Layout Reflow

Changing `top`, `left`, `width`, `height` triggers:

1. **Reflow (layout recalculation)**
2. **Repaint**
3. **Composite**

This is slow.

---

### ‚úî Reason 2 ‚Äî GPU Acceleration

`transform` and `opacity` are handled by the **GPU**:

* No layout changes
* No heavy recalculation
* Browser uses a separate compositing layer

---

### ‚úî Reason 3 ‚Äî Browser Optimizations

Browsers can batch & offload transform animations:

```css
transform: translate();
transform: scale();
transform: rotate();
opacity: 0.5;
```

These are the **fastest properties** to animate.
