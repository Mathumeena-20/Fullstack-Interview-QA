# ✅ **1. What are ways to optimize HTML for fast-loading websites?**

### ✔ 1. Minimize HTML size

* Remove unnecessary comments
* Remove whitespace
* Use HTML minification
* Reduce nested elements

---

### ✔ 2. Use semantic HTML

Browsers parse semantic HTML faster because structure is predictable.

---

### ✔ 3. Load scripts efficiently

* Use `defer` or `async`
* Place non-critical scripts at the end of body
  Example:

```html
<script src="app.js" defer></script>
```

---

### ✔ 4. Preload important resources

Fonts, hero images, critical JS:

```html
<link rel="preload" href="main.css" as="style">
```

---

### ✔ 5. Use lazy loading for images

Load images only when they appear in the viewport.

```html
<img src="img.jpg" loading="lazy">
```

---

### ✔ 6. Use CDN for static assets

Reduces latency and improves caching.

---

### ✔ 7. Optimize images

* Use WebP/AVIF
* Compressed images
* Correct dimensions

---

### ✔ 8. Reduce DOM size

Faster parsing and rendering.

---

### ✔ 9. Use caching (HTTP cache headers)

---

### ✔ 10. Avoid inline CSS/JS when possible

Improves caching and reduces HTML size.

---

# ✔ Interview summary:

HTML optimization = faster parsing + fewer blocking resources + optimized images + preload + lazy loading.

# ✅ **2. Why should you minimize DOM depth?**

Deep DOM trees make the browser work harder.

### Problems with deep DOM:

1. **Slower DOM parsing**

   * More nodes → slower load

2. **Slower CSS Selector Matching**

   * Browser checks selectors from right-to-left
   * Deep trees = expensive lookup

3. **Slower Layout & Reflow**

   * Small changes require recalculation for many nodes

4. **More memory usage**

---

### Example of bad (deep nesting):

```html
<div>
  <div>
    <div>
      <div>
        <p>Text</p>
      </div>
    </div>
  </div>
</div>
```

### Good:

```html
<p>Text</p>
```

---

# ✔ Interview summary:

> Shallow DOM = faster parsing, faster CSS, faster rendering, better performance.

# ✅ **3. What is lazy loading?**

### **Definition:**

Lazy loading means **loading images, videos, or iframe content only when they enter the viewport** (i.e., when the user scrolls to them).

This improves:

* Page load speed
* Saves bandwidth
* Improves Core Web Vitals (LCP, FCP)

### Example (native HTML5 lazy loading):

```html
<img src="product.jpg" loading="lazy" alt="Product image">
```

### Lazy loading for iframes:

```html
<iframe src="video.html" loading="lazy"></iframe>
```

---

# ✔ Interview summary:

> Lazy loading avoids loading off-screen resources until needed, making websites faster.

# ✅ **4. How do you preload critical resources? Example using `<link rel="preload">`.**

### **Definition:**

Preloading tells the browser to download certain critical assets early, even before they are discovered during parsing.

Used for:

* Critical CSS
* Fonts
* Hero images
* Main JavaScript
* Background banners

---

### Example: Preload CSS

```html
<link rel="preload" href="styles.css" as="style">
```

### Example: Preload Font

```html
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
```

### Example: Preload a hero image

```html
<link rel="preload" href="hero.jpg" as="image">
```

### Example: Preloading JS

```html
<link rel="preload" href="main.js" as="script">
```

---

# ✔ Interview summary:

> Preload tells the browser “this file is important — download it NOW” before rendering starts.

# ✅ **5. What is the Critical Rendering Path?**

### **Definition:**

The Critical Rendering Path (CRP) is the sequence of steps the browser takes to convert HTML, CSS, and JS into **pixels on the screen**.

### **Steps in Critical Rendering Path:**

1. **HTML → DOM**
   Browser parses HTML and builds the DOM tree.

2. **CSS → CSSOM**
   Browser parses CSS and builds CSSOM tree.

3. **DOM + CSSOM → Render Tree**
   Determines what is visible and how it should look.

4. **Layout / Reflow**
   Browser calculates positions and sizes of elements.

5. **Painting**
   Browser fills pixels on the screen.

---

### Why is it important?

Blocking resources (like CSS files or scripts without `async/defer`) delay these steps → delay first render → slow site.

---

### Example of blocking CSS:

```html
<link rel="stylesheet" href="style.css">
```

CSS **blocks rendering** until loaded.

### Non-blocking JS:

```html
<script src="app.js" defer></script>
```
