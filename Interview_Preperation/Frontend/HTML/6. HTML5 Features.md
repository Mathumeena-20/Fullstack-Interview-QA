# ✅ **2. Purpose of `<canvas>` and `<svg>`**

## **<canvas>**

* Used for **drawing shapes**, **animations**, **games**, **dynamic graphics** using JavaScript
* Resolution depends on pixel size
* Good for **real-time graphics**

### Example:

```html
<canvas id="myCanvas" width="200" height="100"></canvas>

<script>
  const c = document.getElementById("myCanvas");
  const ctx = c.getContext("2d");
  ctx.fillStyle = "red";
  ctx.fillRect(20, 20, 150, 60);
</script>
```

---

## **<svg>**

* Vector-based graphics
* Scales without losing quality
* Declarative (no JavaScript needed for basic shapes)

### Example:

```html
<svg width="200" height="100">
  <circle cx="50" cy="50" r="40" fill="blue"></circle>
</svg>
```

---

## **Difference Summary**

| Feature  | `<canvas>`                | `<svg>`                 |
| -------- | ------------------------- | ----------------------- |
| Type     | Pixel-based               | Vector-based            |
| Quality  | Loses quality when zoomed | Scales perfectly        |
| DOM      | Not part of DOM           | Elements are in DOM     |
| Use case | Games, animations         | Icons, charts, diagrams |

# ✅ **3. How Does the Drag and Drop API Work?**

HTML5 provides built-in **drag-and-drop** functionality.

### Steps:

---

## **1. Make an element draggable**

```html
<div draggable="true" id="item">Drag Me</div>
```

---

## **2. Listen to drag events**

```javascript
item.addEventListener("dragstart", (e) => {
  e.dataTransfer.setData("text/plain", e.target.id);
});
```

---

## **3. Allow drop on target**

```html
<div id="dropZone"></div>

<script>
  dropZone.addEventListener("dragover", (e) => e.preventDefault());
```

---

## **4. Handle drop**

```javascript
dropZone.addEventListener("drop", (e) => {
  e.preventDefault();
  const id = e.dataTransfer.getData("text");
  const el = document.getElementById(id);
  dropZone.appendChild(el);
});
</script>
```

---

### ✔ Interview Summary

* `draggable="true"` enables dragging
* `dragstart` stores data
* `dragover` must call `e.preventDefault()`
* `drop` handles the drop action

# ✅ **4. Difference Between `<audio>` and `<video>` Controls**

Both support:

* `controls`
* `autoplay`
* `loop`
* `muted`

---

## **Audio Example**

```html
<audio controls>
  <source src="song.mp3" type="audio/mpeg">
</audio>
```

---

## **Video Example**

```html
<video controls width="350">
  <source src="movie.mp4" type="video/mp4">
</video>
```

---

## **Key Differences**

| Feature    | `<audio>`             | `<video>`                                         |
| ---------- | --------------------- | ------------------------------------------------- |
| Visual UI  | Minimal player        | Full video player                                 |
| Display    | No screen             | Shows video element                               |
| Attributes | Mostly audio controls | Additional attrs like `poster`, `width`, `height` |

---

# ✔ Interview Summary:

Use `<audio>` for sound, `<video>` for video with UI support.

# ✅ **5. What is Web Storage API? Explain Use Cases.**

HTML5 Web Storage API provides:

### **1. localStorage**

* Permanent storage
* Does not expire
* 5–10 MB capacity
* Good for **settings, saved items**

---

## **Example: localStorage**

```javascript
localStorage.setItem("theme", "dark");
localStorage.getItem("theme");
```

---

### **2. sessionStorage**

* Lives only for **browser tab session**
* Clears when tab closes
* Good for **temporary data**

---

## **Example: sessionStorage**

```javascript
sessionStorage.setItem("token", "123456");
```

---

### Use Cases

| Use Case                              | Storage        |
| ------------------------------------- | -------------- |
| Save user theme                       | localStorage   |
| Save cart items                       | localStorage   |
| Save form progress temporarily        | sessionStorage |
| Keep user logged in while tab is open | sessionStorage |
| Save scroll position or UI state      | sessionStorage |
