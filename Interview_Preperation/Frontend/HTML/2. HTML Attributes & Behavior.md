# ‚úÖ **1. Difference Between `id` and `class`**

| Feature     | `id`                                 | `class`                             |
| ----------- | ------------------------------------ | ----------------------------------- |
| Uniqueness  | Must be **unique** in the whole page | Can be reused multiple times        |
| Usage       | Identify a single element            | Group multiple elements             |
| CSS         | `#id` selector                       | `.class` selector                   |
| JS          | `document.getElementById()`          | `document.getElementsByClassName()` |
| Recommended | For unique elements (header, main)   | For styling reusable components     |

### Example:

```html
<div id="main-header">Welcome</div>

<div class="card">Product 1</div>
<div class="card">Product 2</div>
```

# ‚úÖ **2. What is the use of `data-attributes`? Give Example**

### **Definition**

`data-*` attributes are used to store **extra information** in HTML elements that JavaScript can read.

### Why useful?

* Pass dynamic data to JavaScript
* Store temporary values
* No need for additional hidden fields
* Helps in analytics, event tracking, dynamic UI

### Example (Button with product info):

```html
<button 
  class="add-to-cart"
  data-product-id="101"
  data-price="499"
>
  Add to Cart
</button>
```

### Reading in JavaScript:

```javascript
const btn = document.querySelector('.add-to-cart');
console.log(btn.dataset.productId); // "101"
console.log(btn.dataset.price);     // "499"
```

# ‚úÖ **3. Difference Between `href="#"` and `href="javascript:void(0)"`**

### ‚úî `href="#"`

* Navigates to the **top of the page**
* Causes the page to jump
* Not good for buttons
* Used when linking to an element with `id`

Example:

```html
<a href="#contact">Go to Contact</a>
```

### ‚úî `href="javascript:void(0)"`

* Does **NOT reload** or jump
* Does **NOT** navigate anywhere
* Used when link acts as a button (with JS handlers)

Example:

```html
<a href="javascript:void(0)" onclick="openModal()">Open Modal</a>
```

### Interview Summary:

* `#` ‚Üí scrolls to the top
* `javascript:void(0)` ‚Üí no page movement
* Better option today:

  ```html
  <a href="#" onclick="return false;">Click</a>
  ```

  OR
  **Use `<button>` instead of `<a>` if no navigation is needed.**

# ‚úÖ **4. Difference Between `defer` and `async` in Script Tags**

Both load external scripts **without blocking HTML parsing**, but execution timing differs.

---

## **async**

* Loads script in parallel with HTML parsing
* Executes **immediately when downloaded**
* Execution order is **NOT guaranteed**
* Good for:

  * Analytics scripts
  * Ads
  * Third-party widgets

Example:

```html
<script src="analytics.js" async></script>
```

---

## **defer**

* Loads script in parallel
* Executes **after HTML is fully parsed**
* Execution order **is maintained**
* Best for:

  * Main application scripts
  * DOM-dependent code

Example:

```html
<script src="app.js" defer></script>
```

---

## Summary Table

| Attribute | When Script Loads | When Script Executes  | Order Preserved? |
| --------- | ----------------- | --------------------- | ---------------- |
| **async** | Parallel          | As soon as downloaded | ‚ùå No             |
| **defer** | Parallel          | After DOM ready       | ‚úî Yes            |

## **`<meta charset="UTF-8">`**

* Sets the **character encoding** for the webpage.
* UTF-8 supports:

  * English
  * Emojis
  * Symbols
  * All major world languages

Without this, some characters may show as ‚ÄúÔøΩ‚Äù.

Example:

```html
<meta charset="UTF-8">
```

---

## **`<meta name="viewport" content="width=device-width, initial-scale=1.0">`**

* Controls how the page looks on **mobile devices**.
* `width=device-width` ‚Üí Page width matches device width
* `initial-scale=1.0` ‚Üí No zoom applied initially

Without this meta tag:

* Website looks zoomed-out on mobile
* Text becomes very small
* Layout does not behave responsively

Example:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
