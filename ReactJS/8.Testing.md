# âœ… **2. What is React Testing Library (RTL)?**

# ================================

React Testing Library is a **lightweight testing library** for testing React components.

### Its philosophy:

> Test components as real users interact with them â€” NOT by testing implementation details.

### Why use RTL?

âœ” Encourages accessibility (byRole, byLabelText, etc.)
âœ” Simulates real user interactions
âœ” Avoids brittle tests
âœ” Works with Jest

---

### Example RTL component test:

```jsx

import userEvent from "@testing-library/user-event";
import Button from "./Button";

test("button click", () => {
  render(<Button />);

  userEvent.click(screen.getByRole("button"));

  expect(screen.getByText("Clicked")).toBeInTheDocument();
});
```

### â­ Unit Testing

# âœ… **5. How to Test Custom Hooks**

# ================================

React Testing Library provides:

```js

```

### Example hook:

```jsx

  const [count, setCount] = useState(0);
  return { count, increment: () => setCount(c => c + 1) };
}
```

### â­ Hook test:

```js
import { renderHook, act } from "@testing-library/react";

test("useCounter increments", () => {
  const { result } = renderHook(() => useCounter());

  act(() => {
    result.current.increment();
  });

  expect(result.current.count).toBe(1);
});
```

âœ” Hooks must be wrapped inside `act`
âœ” `renderHook` gives access to hook return values

# âœ… **6. How to Test API Calls**

# ================================

We **mock fetch or axios** to avoid calling real APIs.

Example component:

```jsx

  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("/api/users")
      .then(res => res.json())
      .then(data => setUsers(data));
  }, []);

  return <div>{users.length} users</div>;
}
```

---

### â­ Mock fetch in Jest:

### Test:

```js
  expect(await screen.findByText("2 users")).toBeInTheDocument();
});
```

âœ” `findByText` waits for async rendering
âœ” No real API call is made
