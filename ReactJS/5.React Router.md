# ‚úÖ **1. How does client-side routing work?**

In traditional web apps:
‚û° Clicking a link ‚Üí browser sends HTTP request ‚Üí server responds with a new HTML page.

In React (client-side routing):
‚û° **The page does NOT reload.**
‚û° React intercepts navigation and changes **the UI based on the URL**
‚û° The entire app is usually loaded once (SPA ‚Äì Single Page Application)

### ‚≠ê How React Router works:

1. Listens to URL changes using **History API** (`pushState`, `popstate`)
2. Matches the current URL to a **route**
3. Renders the associated component
4. Doesn‚Äôt reload the page ‚Üí faster & seamless user experience

### Example:

```jsx
<Routes>
  <Route path="/home" element={<Home />} />
  <Route path="/about" element={<About />} />
</Routes>
```

When user visits `/about`, React Router renders `<About />` **without reloading the page**.

# ‚úÖ **2. Difference Between `<BrowserRouter>` vs `<HashRouter>`**

| Feature                 | BrowserRouter         | HashRouter                   |
| ----------------------- | --------------------- | ---------------------------- |
| URL format              | `/about`              | `#/about`                    |
| Uses                    | **HTML5 history API** | **hash portion of URL**      |
| Requires server config? | YES                   | NO                           |
| SEO                     | Better SEO            | Not SEO friendly             |
| When to use?            | Real apps, SEO pages  | GitHub Pages, static hosting |

---

### ‚úî BrowserRouter Example:

```jsx
import { BrowserRouter } from "react-router-dom";

<BrowserRouter>
  <App />
</BrowserRouter>
```

### ‚úî HashRouter Example:

```jsx
import { HashRouter } from "react-router-dom";

<HashRouter>
  <App />
</HashRouter>
```

---

### üéØ Interview summary:

> BrowserRouter is modern and clean but needs server config.
> HashRouter works anywhere because it uses URL hash and does NOT require server support.

# ‚úÖ **3. How to redirect programmatically in React Router?**

In React Router v6:

### ‚úî Use `useNavigate()` hook

```jsx
import { useNavigate } from "react-router-dom";

function Login() {
  const navigate = useNavigate();

  function handleLogin() {
    // After login success
    navigate("/dashboard");
  }

  return <button onClick={handleLogin}>Login</button>;
}
```

### You can also replace history:

```jsx
navigate("/dashboard", { replace: true });
```

---

### üéØ Interview summary:

> React Router uses `useNavigate()` for programmatic redirects.

# ‚úÖ **4. What are nested routes?**

Nested routes let you define parent/child UI layouts.

Example:
Dashboard ‚Üí Users ‚Üí User Details

---

### ‚≠ê Example of nested routes:

```jsx
<Routes>
  <Route path="/dashboard" element={<DashboardLayout />}>
    <Route path="users" element={<Users />} />
    <Route path="settings" element={<Settings />} />
  </Route>
</Routes>
```

In `DashboardLayout`:

```jsx
function DashboardLayout() {
  return (
    <div>
      <Sidebar />
      <Outlet /> {/* Nested route renders here */}
    </div>
  );
}
```

### How it works:

* Visiting `/dashboard/users` renders:

  * DashboardLayout
  * Users component inside `<Outlet />`

---

### üéØ Interview summary:

> Nested routes allow building layouts where child components render inside a parent, using `<Outlet />`.

# ‚úÖ **5. What is lazy loading in React Router?**

Lazy loading = loading components only when needed ‚Üí improves performance.

React Router works with React‚Äôs `lazy()` + `Suspense`.

---

### ‚≠ê Example:

```jsx
import { lazy, Suspense } from "react";

const About = lazy(() => import("./About"));

<Routes>
  <Route 
    path="/about" 
    element={
      <Suspense fallback={<p>Loading...</p>}>
        <About />
      </Suspense>
    }
  />
</Routes>
```

‚úî About page loads **only when user visits /about**
‚úî Decreases initial bundle size ‚Üí faster load times

---

### üéØ Interview summary:

> Lazy loading splits code into chunks so React Router loads components only when a route is visited.
